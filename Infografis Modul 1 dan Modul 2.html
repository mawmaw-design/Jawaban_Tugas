Modul 1 • Bagian A • Tugas 1

Skrining Awal Pasien Demam

1. Algoritma Skrining Awal (Teks Utuh)

1. Mulai program.

2. Input Data Pasien:

  • Nilai angka: Suhu tubuh (°C), Saturasi oksigen (%), Tekanan darah sistolik (mmHg), dan Trombosit (/mm³).
  • Jawaban Ya/Tidak: Batuk, Sesak napas berat, Nyeri otot, Ruam merah, dan Muntah terus-menerus.

3. Evaluasi 1: Penyakit Berat / Kondisi Kritis

  • JIKA Saturasi < 90% ATAU Tekanan Darah < 90 mmHg ATAU Muntah terus-menerus ATAU Sesak berat ATAU Trombosit sangat rendah
  • MAKA Keputusan = RUJUK ke rumah sakit (Kondisi Kritis).

4. Evaluasi 2: COVID-19 Sedang (Jika Evaluasi 1 tidak terpenuhi)

  • JIKA Suhu demam DAN Batuk = Ya DAN Saturasi 90-94% DAN Sesak berat = Ya
  • MAKA Keputusan = RUJUK ke rumah sakit (COVID-19 Sedang).

5. Evaluasi 3: Demam Berdarah / DBD (Jika Evaluasi 1 & 2 tidak terpenuhi)

  • JIKA Suhu > 39°C DAN Nyeri otot = Ya DAN Ruam = Ya DAN Trombosit < 100.000
  • MAKA Keputusan = RAWAT INAP (Monitoring cairan dan trombosit).

6. Evaluasi 4: Demam Biasa (Jika Evaluasi 1, 2, & 3 tidak terpenuhi)

  • JIKA Suhu 37.5 - 38.5°C DAN Saturasi Normal DAN Sesak = Tidak DAN Trombosit Normal
  • MAKA Keputusan = RAWAT JALAN (Istirahat dan obat simptomatik).

7. Kondisi Lainnya: Jika tidak masuk ke kategori di atas, Keputusan = OBSERVASI MEDIS.

8. Tampilkan Hasil Keputusan.

9. Selesai.

Modul 1 • Bagian B • Tugas 2

Prediksi Perburukan & CDSS

1. Algoritma CDSS dan Prediksi Perburukan

1. Mulai program.

2. Inisialisasi Variabel: Set nilai total_skor = 0.

3. Input Data: Baca parameter Suhu, Saturasi, Tekanan Darah Sistolik, Nadi, Laju Napas, Trombosit, Sesak, Muntah, Batuk, Nyeri, dan Ruam.

4. Tahap 1: Identifikasi Penyakit Awal

Lakukan skrining awal (Demam Biasa, COVID, DBD, Kritis) sesuai algoritma Tugas 1. Simpan status ini.

5. Tahap 2: Penilaian Risiko Perburukan (Perhitungan Skor)
(Sistem akan mengecek parameter satu per satu dan menjumlahkan skornya secara kumulatif)

  • JIKA Suhu > 39.0°C, MAKA total_skor = total_skor + 1.
  • JIKA Saturasi < 90%, MAKA total_skor = total_skor + 4. SEBALIKNYA JIKA Saturasi 90 - 94%, MAKA total_skor = total_skor + 2.
  • JIKA Tekanan Darah < 90 mmHg, MAKA total_skor = total_skor + 3.
  • JIKA Nadi > 130 bpm, MAKA total_skor = total_skor + 2.
  • JIKA Laju Napas > 30x/menit, MAKA total_skor = total_skor + 2.
  • JIKA Trombosit < 50.000, MAKA total_skor = total_skor + 4. SEBALIKNYA JIKA Trombosit < 100.000, MAKA total_skor = total_skor + 2.
  • JIKA Sesak napas = Ya, MAKA total_skor = total_skor + 2.
  • JIKA Muntah terus = Ya, MAKA total_skor = total_skor + 1.

6. Tahap 3: Interpretasi dan Rekomendasi Penanganan

  • JIKA total_skor > 8, MAKA Status = Sangat Kritis. Rekomendasi: Rujukan ICU, Oksigen, Cairan infus.
  • JIKA total_skor 6 - 8, MAKA Status = Risiko Tinggi. Rekomendasi: Observasi ketat, Pemeriksaan laboratorium.
  • JIKA total_skor 3 - 5, MAKA Status = Risiko Sedang. Rekomendasi: Lakukan observasi.
  • JIKA total_skor 0 - 2, MAKA Status = Stabil. Rekomendasi: Rawat jalan.

7. Tampilkan Hasil Tahap 1, Tahap 2, dan Tahap 3.

8. Selesai.

Modul 2

Sistem Monitoring ICU Sederhana (Real-Time)

Karena konsep dasar di Tugas 1 dan Tugas 2 Modul 2 ini saling melengkapi, algoritma di bawah adalah versi gabungan/pengembangan yang komprehensif untuk sistem pemantauan kontinu.

1. Algoritma Monitoring ICU Real-Time

1. Inisialisasi Sistem: Siapkan variabel Suhu, SpO2, Sistolik, Diastolik, BPM. Siapkan output LED dan Alarm.

2. Mulai Perulangan Utama (Looping Real-Time)

3. Baca Sensor: Ambil data terbaru secara real-time.

4. Evaluasi Suhu Tubuh:

  • Jika 36 - 37.5°C = NORMAL.
  • Jika 37.6 - 38.5°C = WARNING.
  • Jika > 38.5°C ATAU < 35°C = KRITIS.

5. Evaluasi SpO2:

  • Jika ≥ 95% = NORMAL.
  • Jika 90 - 94% = WARNING.
  • Jika < 90% = KRITIS.

6. Evaluasi Tekanan Darah (Sistolik & Diastolik):

  • Sistolik: NORMAL (90-120), WARNING (121-140), KRITIS (>140 atau <80).
  • Diastolik: NORMAL (60-80), WARNING (81-90), KRITIS (>90 atau <50).

7. Evaluasi Detak Jantung (BPM):

  • Jika 60 - 100 bpm = NORMAL.
  • Jika 101 - 120 bpm = WARNING.
  • Jika > 120 bpm ATAU < 50 bpm = KRITIS.

8. Penentuan Status Keseluruhan:

  • Jika ada minimal satu KRITIS ➔ Status Akhir = KRITIS. Nyalakan Alarm dan LED Merah.
  • Jika tidak ada KRITIS, tapi ada minimal satu WARNING ➔ Status Akhir = WARNING. Nyalakan LED Kuning.
  • Jika semua NORMAL ➔ Status Akhir = NORMAL. Nyalakan LED Hijau.

9. Tampilkan data ke monitor.

10. Jeda Waktu (Delay 1 detik), lalu kembali ke langkah nomor 3.
